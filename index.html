1 Heatmaps of different metrics

Figure 1.1: Heatmaps of normalized Enrichment Factors at different %s for the different programs and systems. Rows (systems) are sliced by type of system, according to the left annotation. Columns (programs) are sorted by decreasing value of average EF across all systems for each %. Normalization of EFs is performed by dividing each EF value by the best possible value that can be obtained for the corresponding % (which depends on the library size and amount of actives in it) and then applying a log10 transformation (0,001 is added to all values to avoid zeroes). 

other_hm <- function(metric, lowerbest=F, ...) {
  wider <- metrics %>%
    select(Type, System, Program, metric) %>%
    pivot_wider(names_from = Program, values_from = metric)
  get_hm(metric, wider, lowerbest, ...)
}
